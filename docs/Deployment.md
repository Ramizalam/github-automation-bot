# Deployment Guide

This project is designed to run seamlessly on Vercel or any modern Node.js hosting provider, paired with a serverless Postgres database like Neon.

## Prerequisites

1. **GitHub OAuth App**: For authenticating users into the dashboard.
   - Go to GitHub Developer Settings → OAuth Apps → New OAuth App
   - Authorization callback URL: `https://yourdomain.com/api/auth/callback/github`
2. **GitHub App (Optional but recommended for strict rate limits)**: Used for taking actions on repositories (adding labels, closing issues).
3. **Neon PostgreSQL Database**: Get a connection string from neon.tech.
4. **Slack App (Optional)**: If you want to use the Slack notification action strategy. Create an incoming webhook URL.

## Environment Variables

Your production environment must have these variables set (matching `.env.example`):

```bash
DATABASE_URL="postgresql://user:pass@ep-restless...neon.tech/neondb?sslmode=require"
NEXTAUTH_URL="https://yourdomain.com"
NEXTAUTH_SECRET="use-a-strong-random-string"

GITHUB_ID="your_oauth_client_id"
GITHUB_SECRET="your_oauth_client_secret"
GITHUB_WEBHOOK_SECRET="the_secret_for_validating_payloads"

SLACK_WEBHOOK_URL="https://hooks.slack.com/services/..."
SLACK_MAX_RETRIES="3"
```

## Vercel Deployment

The fastest way to deploy this Next.js app is on Vercel.

1. Push your code to a GitHub repository.
2. Import the project in the Vercel dashboard.
3. Vercel will automatically detect Next.js and apply the correct build settings (`npm run build`).
4. **Crucial**: Add all Environment Variables from the section above into the Vercel Project Settings *before* triggering the first build.
5. Deploy.

## Database Migrations in Production

When deploying a new database schema using Prisma, do **not** run `npx prisma db push` in production. Instead, generate and deploy migrations.

1. Locally, run: `npx prisma migrate dev --name <migration_name>`
2. Commit the generated `prisma/migrations` folder to Git.
3. In production, run `npx prisma migrate deploy` during the build step.
   - *If using Vercel, update your build command in the dashboard to:* `npx prisma generate && npx prisma migrate deploy && next build`

## Webhook Delivery Configuration

Once deployed, you need to point GitHub webhooks to your live URL.
1. When connecting a repository in the dashboard, the app generates a `webhookSecret`.
2. Go to your repository settings on GitHub → Webhooks → Add Webhook.
3. Payload URL: `https://yourdomain.com/api/webhooks/github`
4. Content type: `application/json`
5. Secret: Paste the secret generated by the dashboard.
6. Select the events you want to listen to (e.g. Pull Requests, Issues, Pushes).
